# Memory Viewer v2 — P6 质量打磨迭代

> **日期**: 2026-05-29
> **目标**: 修复已知问题，提升前端交互质量和生产可用性

---

## 优先级排序

### 🔴 P0 — 必须修复（数据/功能缺陷）

| # | 问题 | 影响 | 修复方案 |
|---|------|------|----------|
| F33 | /api/agentmemory 非分页端点只返回 1 条 | 搜索/导出/图谱等依赖此端点的功能数据不全 | 检查 get_all_memories() 的 archived 过滤逻辑，非分页端点应返回全部（含归档） |
| F34 | API 错误时前端无友好提示 | 用户操作失败后看不到原因 | 全局 axios/fetch 拦截器 + Toast 组件 |
| F35 | MCP 端点无认证 | 任何人可读写记忆 | 添加 API Key 认证（X-API-Key header） |

### 🟡 P1 — 应该修复（体验问题）

| # | 问题 | 影响 | 修复方案 |
|---|------|------|----------|
| F36 | 暗色模式 CSS 覆盖不全 | 部分组件在暗色模式下色差/看不清 | 全局审查所有组件的暗色样式，补充 CSS 变量 |
| F37 | 前端无响应式适配 | 移动端排版错乱 | 添加 media query 断点，卡片/侧边栏/表格响应式 |
| F38 | 版本历史缺少创建时初始版本 | 编辑前无法回溯原始内容 | 创建记忆时自动保存 v1 快照 |
| F39 | 错误状态无空状态设计 | 加载失败/无数据时页面空白 | 添加 ErrorBoundary + 空状态插画/提示 |

### 🟢 P2 — 优化项（功能增强）

| # | 问题 | 影响 | 修复方案 |
|---|------|------|----------|
| F40 | 多实例管理只读配置 | 无法实际切换查看其他实例 | 实现代理请求到目标实例 API |
| F41 | 飞书集成只有 webhook 发送 | 没有交互式操作 | 添加飞书消息卡片按钮回调（可选，复杂度高） |
| F42 | 图谱/时间线在数据少时效果差 | 11 个节点太稀疏 | 添加引导文案 + 示例数据模式 |
| F43 | 健康度评分列表模式不预加载 | 需要展开才能看到 | 批量预加载简化版分数（列表 API 附加 health_score） |

## 开发顺序

```
F33(数据bug) → F34(错误处理) → F35(MCP认证) → F36(暗色模式) → F39(空状态)
→ F37(响应式) → F38(版本初始) → F43(健康度预加载) → F42(引导文案) → F40(多实例代理)
```

F41(飞书交互) 复杂度高，建议延后到 P7。
